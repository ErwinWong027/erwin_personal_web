# 三协同升级计划：首页、AI方法论页面、Before/After对比

## 任务概述

完成三个协同升级，将网站从"变更日志"风格转化为"产品化迭代案例研究"风格，最终呈现效果应像一份优秀的产品经理AI实践案例研究。

---

## 一、首页升级：演进历程板块

### 板块名称
"演进历程"（Evolution Journey）

### 结构设计
```
标题 + 简短介绍
↓
4个阶段卡片（响应式：桌面横排，移动端纵排）
↓
CTA按钮 → 跳转至方法论页面
```

### 四阶段内容（产品语言，无版本号）

#### 阶段1：问题定义 (Problem Framing)
- 核心成果：定义受众、MVP验证
- 简述：从需求分析到技术选型，明确目标受众为招聘官/面试官

#### 阶段2：数据驱动 (Data-Grounded)
- 核心成果：结构化数据、去模版化
- 简述：用真实数据替换占位内容，提升可信度

#### 阶段3：AI原生交互 (AI-Native)
- 核心成果：AI助手、智能问答
- 简述：加入基于个人数据的AI交互功能

#### 阶段4：产品化演进 (Productization)
- 核心成果：架构优化、体验打磨
- 简述：信息架构精简，导航与交互优化

### 视觉设计
- 使用卡片或时间轴形式
- 每个阶段包含：图标、标题、核心成果标签、简述
- 渐变色彩区分阶段
- 底部CTA按钮："深入了解构建过程 →"

### 实现步骤
1. 创建 `src/components/sections/IterationSummary.tsx`
2. 在 `zh.json` 添加 `iterationSummary` 翻译
3. 在 `en.json` 添加 `iterationSummary` 翻译
4. 修改 `src/app/[locale]/page.tsx`，在 `<About />` 后添加组件

## 二、AI方法论页面重构

### 页面结构

#### A. Hero区域
- **标题**：网站迭代过程 / Website Iteration Process
- **副标题**：From MVP to polished product
- **简述**：关于"人机协作构建"的核心理念
- **设计**：使用现有的 `SectionHeading` 组件

#### B. 核心逻辑板块
阐述本站的构建哲学：
- **受众先行**：从招聘官视角设计信息架构
- **信息结构化**：JSON数据驱动内容生成
- **AI协同迭代**：人机对话驱动的持续优化

#### C. 四阶段详情展开

每个阶段包含：
- **背景**：为什么需要这个阶段
- **变更内容**：具体做了什么
- **价值所在**：带来了什么改进
- **阶段产出**：具体的交付物

##### 阶段1：问题定义 (Problem Framing)

**背景**：
- 从作业/任务需求出发
- 需要进行需求分析、竞品研究、素材组织

**变更内容**：
1. **AI协作流程**：
   - 首先使用 Gemini 帮助定义目标并生成 Prompt
   - 通过多轮对话，确定技术栈：Next.js、Tailwind CSS、集成组件库
   - 随后转向 Trae IDE，配合 GLM 5 和 Qwen 模型生成计划

2. **目标定义**：
   - 定义目标受众：招聘官 / 简历筛选者
   - 定义预期体验：极简、专业、高信任感

3. **规划输出**：
   - 要求 AI 输出：板块设计、信息层级、UX 流程
   - 先构建 MVP（最小可行性产品）计划，评估优缺点，并完善计划

4. **工具迁移**：
   - 重写 Prompt，迁移到 Cursor 使用 Claude
   - 添加来自社交媒体的个人网站参考作为视觉方向输入

**价值所在**：
- 明确目标受众和体验定位
- 建立清晰的技术栈和架构方向
- 通过MVP验证降低风险

**阶段产出**：
- MVP方案
- 技术栈选型（Next.js + Tailwind CSS + 组件库）
- 板块设计、信息层级、UX流程图

---

##### 阶段2：数据驱动构建 (Data-Grounded Build)

**背景**：
- 首版使用通用占位内容，缺乏个性化
- 需要提升可信度和真实性

**变更内容**：
1. **数据结构化**：
   - 创建模拟的结构化个人资料文件 `mockProfile.json`
   - 包含：教育经历、实习、项目成果、技能标签等

2. **内容替换**：
   - 使用结构化数据替换通用的占位符内容
   - 使网站更具可信度和个性化
   - 根据实际内容的契合度继续打磨细节

3. **功能实现**：
   - 构建第一个完整的运行版本
   - 包含：双语路由、暗黑模式、模块化首页
   - 通过自然语言交互生成多个源文件

**价值所在**：
- 提升可信度和个性化
- 数据驱动内容生成，便于维护
- 建立可复用的数据结构

**阶段产出**：
- `mockProfile.json` 结构化数据源
- 首个可运行版本
- 双语路由、暗黑模式、模块化首页

---

##### 阶段3：AI原生交互 (AI-Native Interaction)

**背景**：
- 静态展示无法满足深度了解需求
- 需要提供更智能的交互方式

**变更内容**：
1. **功能创新**：
   - 在首页添加创新的 AI 聊天功能
   - AI 助手基于提供的个人资料数据，回答有关作品集所有者的问题

2. **技术实现**：
   - 使用 Qwen-Plus API
   - 通过与 AI 对话，逐个模块构建聊天功能
   - 生成多个源文件：知识库、System Prompt、API 路由、聊天 UI

3. **范围约束**：
   - 助手范围限定在作品集内容内
   - 避免"幻觉"问题

4. **视觉优化**：
   - 优化 AI 聊天 UI：机器人图标
   - 更新欢迎语和推荐问题
   - 移除不必要的元素
   - 添加打字机样式的欢迎交互

**价值所在**：
- 提供智能问答能力
- 增强用户参与度和交互体验
- 展示AI原生应用能力

**阶段产出**：
- AI 助手功能
- 知识库、System Prompt、API 路由
- 优化的聊天 UI

---

##### 阶段4：产品化与架构精炼 (Productization & Architecture Refinement)

**背景**：
- 功能扩展后信息架构需优化
- 单页展示无法承载长篇内容

**变更内容**：
1. **功能扩展**：
   - 添加画廊 (Gallery) 板块
   - 添加经历时间轴 (Timeline)

2. **架构重组**：
   - 首页保留核心板块
   - 时间轴 / AI 方法论 / 联系方式变为独立页面

3. **导航优化**：
   - 首页带有子板块的下拉菜单
   - 为长篇内容设置专用页面

4. **优先级调整**：
   - 将 AI 助手在首页的位置提前，提高交互优先级
   - 优化语言切换器
   - 将项目和画廊展示合并为统一的"项目展示"方向

5. **持续改进**：
   - 持续改进布局、层级和阅读流

**价值所在**：
- 优化信息架构，提升长内容阅读体验
- 提高核心功能的可见性和交互优先级
- 建立可扩展的产品架构

**阶段产出**：
- 产品化架构（首页 + 独立页面）
- 导航优化（下拉菜单）
- 画廊、时间轴功能

## 三、写作风格与双语要求

### 写作风格
- **简洁**：避免冗长描述，突出核心信息
- **客观**：基于事实描述，避免主观夸大
- **产品意识**：使用产品语言（如"定义受众"、"MVP验证"、"信息架构"）
- **严禁营销词汇**：不使用"颠覆性"、"赋能"、"革命性"等过度包装词汇

### 双语要求
- **中文**：地道自然，符合中文表达习惯
- **英文**：专业规范，避免机翻感
- **一致性**：中英文内容对应，术语统一
